A developer is working on a `feature` branch while new changes have been pushed to the `master` branch. The goal is to:

* Update the `feature` branch with the latest changes from `master`
* **Preserve all commits from the `feature` branch**
* **Avoid creating any merge commits**
* Push the final result to the remote repository

## ğŸ“ Environment

* **Remote Repository:** `/opt/games.git`
* **Local Repository:** `/usr/src/kodekloudrepos`
* **Active Branch:** `feature`

## âœ… Step-by-Step Solution

### 1. Check Available Branches

Make sure you are on the `feature` branch:

```bash
git branch
```

Expected output:

```
* feature
  master
```

---

### 2. Rebase `feature` onto `master`

This applies all commits from `feature` on top of the latest commits from `master`.

```bash
git rebase master
```

---

### 3. Verify Repository Status

```bash
git status
```

Expected output:

```
On branch feature
nothing to commit, working tree clean
```

---

### 4. Push Changes to Remote

```bash
git push origin feature
```

If you receive a **non-fast-forward** error, it means the remote branch contains commits that your local branch does not yet have.

---

### 5. Rebase with Remote Branch

To keep a clean, linear history and avoid merge commits, pull using rebase:

```bash
git pull --rebase origin feature
```

---

### 6. Final Push

After a successful rebase, push the branch again:

```bash
git push origin feature
```

---

## ğŸ‰ Result

* The `feature` branch is now fully up to date with `master`
* No merge commits were created
* All feature branch commits are preserved
* Changes are successfully pushed to the remote repository
